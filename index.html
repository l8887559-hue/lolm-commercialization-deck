龙魂水晶

活跃任务 · 熟练度 · 活动

设计意图：强化 BP 的长期回报感，提升续购率。

06 核心问题诊断 · 结构清晰但呈现乱的三个根因 对应用户反馈："玩家理解成本高"

① 多对多投放，认知地图缺失

28 个外观内容 × 9 条渠道 = 典型的多对多关系。 玩家无法建立"我想要这个东西 → 该去哪里获得"的清晰认知。 比如回城特效既在商城、BP、战利品、限定抽奖都有。

商业化后果：用户在"想买但找不到"的摩擦中流失意愿，潜在付费被稀释。

② 品级感知模糊（端游沿用病）

品级划分沿用端游，但手游用户看到 79/99/¥100 原石的皮肤时， 特效感知差异没拉开。 典型如大元素使拉克丝放在一堆传说里并不突出。

商业化后果：高品级溢价空间被自己稀释，ARPPU 天花板被自己压低。

③ 免费货币链过长

龙魂水晶 → 橙色原石 → 银色星尘 → 传说皮肤碎片 → 铸造皮肤， 4 步嵌套转换，每一步都有换算率差异。 玩家算不清值不值，干脆不参与。

商业化后果：免费侧留存工具失效，反哺不到付费。
07 优化点沙盒 · 可探索的商业化维度 对应 JD 第 4 条 · 挖掘商业化形式
前置约束 · 运营 5 年产品的优化边界

LOLM 已运营多年，玩家认知已形成、历史数据已沉淀、历代系统相互耦合。 任何优化方案都要满足三条约束： ① 不打破老用户既有认知（否则反噬留存）； ② 不推翻历史资产价值（不能让已付费用户感觉吃亏）； ③ 增量而非替换（新机制叠加在旧体系之上，老系统继续运转）。 下面三个方向都遵循这个框架。

◆ 针对问题①｜多对多投放的本质：付费意愿过度集中在皮肤

核心洞察：多对多不是渠道设计问题，是玩家付费意愿结构问题—— 付费意愿基本集中在皮肤上，即使开新渠道也不得不捆绑皮肤才能卖得动。 所以唯一的解法是增加道具种类，挖掘潜在消费意愿，把付费重量从皮肤单点分散到多个品类。

① 延伸"皮肤动作姿态"这类新品类尝试

LOLM 已经做了皮肤动作姿态的尝试，效果如果验证过，可以延伸出更多同逻辑品类。 方向：把外观消费从"皮肤本体"扩展到"皮肤的情境化附加表达"。

② 扩展"玩家选择时展示"的系统（对标 Apex 铭牌体系）

现在 LOLM 的名片框 / 铭牌主要只在对局开始 loading 展示一次，而且需要玩家主动点击才能展示， 展示密度低 = 付费意愿无法沉淀。

参考 Apex 的身份展示层：多层边框 + 附加铭牌 + 战绩徽章 + 姿态海报组合， 把身份展示做成玩家日常反复触达的场景，每一层都可以拆成独立付费位。

同时探索：在选取英雄时把这部分内容同步过去（作为皮肤动作姿态的延伸）， 让身份展示从"只在 loading 出现一次"拓展到"选人阶段每局可见"。

对 LOLM 的落地方向：把目前的名片框 / 头像 / 结算表现 / 段位徽章等零散要素， 组装成一套"召唤师个人身份卡"，在大厅、匹配界面、选人页面、对局 loading、结算页面反复调用。 每一层（主海报 / 战绩数据 / 徽章 / 附加铭牌 / 背景底纹）都做成独立付费位。

◆ 针对问题②｜品级感知：常规解已在执行，暂无显著更优方案

现行方案（端游、LOLM 都在用）：在原品级之上增加新品级，用"稀有度跳变"拉动认知。 例如传说 → 神话 → 至臻 → 臻藏 → 典藏 → 终极，每次在顶端加一档。

判断：这个方案没有明显问题。手游的"情境分层（节日版/联动版）"和"品级标签 UI 升级"都已在游戏中落地。 "皮肤动态成长"理论上可行但实操性不高（重做特效成本 ≈ 做新皮肤）。

当前状态：品级感知问题的核心矛盾——品级数量 vs 感知区分度——是端游遗留问题， 暂时没有颠覆性解法，更多是逐步通过 UI 升级 / 情境包装做渐进优化。

影响维度：高品级溢价空间 当前策略：渐进式 UI / 包装升级 颠覆解法：暂无
◆ 针对问题③｜免费货币链过长：简化

现状：龙魂水晶 → 橙色原石 → 银色星尘 → 传说皮肤碎片 → 铸造皮肤，4 步嵌套。 每步都有换算率差异，玩家算不清值不值，干脆不参与。 金铲铲的橙色精粹已经算长了，LOLM 的链路更长。

简化方向：参考金铲铲的做法，但再往"轻"一档。 活跃向玩家的货币路径应该是：

对局积累 + 活动发放 → 直接兑换皮肤

运营 5 年产品的兼容方案：不推翻老链路（老玩家手里还有大量历史货币）， 而是加一条新的"短路径"并行——让新用户默认走短路径，老用户可选保留旧路径兑换历史资产。 给 6 个月过渡期，然后把长链路做淡化处理。

影响维度：免费侧留存反哺付费 技术成本：低-中 付费段受益：F / M 用户
◆ 补充方向④｜扩展 BP 品类：多赛道通行证

参考端游（普通 BP + 乱斗 BP）和金铲铲（3 个 BP 并行）， LOLM 当前只有"峡谷宝典"一个 BP。可根据娱乐模式 / 活动类型增加包装独立 BP。 中小 R 付费活动已证明"活跃 × 付费"绑定模型在 LOLM 生效良好，BP 本身就是这个模型最成熟的载体。

影响维度：付费渗透率 技术成本：低（复用现有 BP 框架） 付费段受益：F→M 转化 · M 留存
◆ 补充方向⑤｜扩展用户触达：全场景"我想要"网络

核心思路：把现有"我想要"功能从仅限商城内扩展到对局前 / 中 / 后全链路， 给玩家在每个接触皮肤的场景都提供标记机会。 本质是把"社交羡慕"这一瞬间的冲动付费意愿捕捉并延后兑现—— 玩家当下心动，但商城入口复杂容易遗忘；全场景标记机制帮玩家记住，活动时精准触发。
